# 功能总览

这份文档用于从产品视角总结 Scholar Blog Studio 当前版本已经具备的能力。

## 1. 项目定位

这个项目现在最适合被理解为三种产品形态的组合：

- 一个学术风格个人网站
- 一个轻量内容管理系统
- 一个带 AI 工具的研究工作台

## 2. 前台能力

### 首页与个人资料

- 学术风格首页
- 个人资料、简介、联系方式展示
- 明暗主题切换
- 响应式布局与侧边导航

### 内容发布

- 博客文章
- 常青笔记
- 日志
- 内容系列 / 专题合集
- 定时发布
- 修订历史与回滚

### 内容发现与导航

- 归档页
- 标签页
- 分类页
- 系列页
- 相关文章推荐
- 全站搜索
- RSS 与 JSON Feed
- sitemap 与 robots 输出

## 3. 账号与互动能力

### 认证闭环

- 注册
- 登录
- 登出
- 邮箱验证
- 找回密码
- 重置密码
- 修改密码
- 敏感变更后的会话失效

### 评论系统

- 登录后发表评论
- 可选的“邮箱验证后才能评论”门槛
- 单用户评论限流
- 垃圾评论启发式检测与重复评论检测
- 评论审核队列
- 基于白名单 / 黑名单规则的自动审核
- 楼中回复
- 管理员删评与审核操作

### 通知闭环

- 评论生命周期邮件通知
- `/account/notifications` 下的站内通知中心
- 导航栏未读计数

## 4. 研究工作流能力

### Papers 模块

- 管理员维护 arXiv 主题
- 手动或定时同步论文
- 公共论文页
- 登录用户个人论文库
- 阅读状态管理（`TO_READ`、`READING`、`COMPLETED`、`ARCHIVED`）
- 阅读进度百分比
- 最近阅读时间
- 批注与高亮摘录

### Weekly Digest

- 基于论文同步数据生成周报
- 公共 Digest 页面
- 后台 Digest 管理
- Digest 的引用与 BibTeX 导出

### 知识连接

- 笔记反向链接
- 引用文本导出
- Papers / Digest 的 BibTeX 导出

## 5. 后台与运维能力

### 内容管理

- 文章 CRUD
- 笔记 CRUD
- 日志 CRUD
- Digest 管理页面
- 内容系列管理
- 定时发布字段
- 修订历史恢复流程

### 审核与用户管理

- 评论审核后台
- 评论规则管理
- 用户角色变更
- 禁言 / 解除禁言
- 封禁 / 恢复
- 软删除 / 恢复
- 会话撤销
- 审计日志

### 分析与导出

- 分析面板
- 数据导出接口与后台导出页
- 管理后台总览卡片

## 6. AI 与 RAG 能力

### Provider 管理

- 多模型提供商注册表
- 模型启停与参数配置
- 可配置语音转写 provider
- API 校验实验室，用于排查 provider 调用问题

### Chat

- 悬浮聊天组件
- 感知当前页面上下文
- 模型切换
- 转写 provider 切换

### RAG

- 手动知识同步
- 混合检索设计
- RAG 管理台
- chunk / source 覆盖与就绪状态查看

## 7. 安全能力

- 自定义会话认证
- HttpOnly Cookie
- 邮箱验证
- 密码重置 token
- 登录限流
- 异常登录提醒
- 管理员 2FA
- 审核与审计控制

## 8. 当前刻意保持轻量的部分

虽然功能已经很多，但有些部分仍然保持“实用优先”而不是企业级复杂度：

- 角色目前仍然只有 `ADMIN` 和 `READER`
- 站内通知中心目前主要覆盖评论与账号相关事件
- RAG 同步仍以手动触发为主
- 导出能力更偏迁移 / 备份，不是多租户报表系统

## 9. 后续可继续强化的方向

如果你还要继续往下做，比较自然的下一步包括：

- 更细粒度的通知偏好设置
- 服务器侧长期后台 worker
- 对象存储与大文件资产管理
- 更强的分析归因能力
- 多模态 RAG 入库
