# 功能总览

这份文档从产品视角总结 Scholar Blog Studio 当前版本已经具备的能力。

## 1. 项目定位

这个项目现在可以理解为四层能力合在一个仓库里：

- 一个公开可访问的学术风格博客站
- 一个以管理员为中心的内容管理后台
- 一个站长私有的研究工作台
- 一套可继续扩展的 AI 与 RAG 能力层

## 2. 前台公开能力

### 首页与个人信息

- 学术风格首页
- 个人简介、头像、链接与联系方式展示
- 明暗主题切换
- 响应式侧边导航

### 内容发布

- 博客文章
- 常青笔记
- Journal 日志
- 图集
- 内容系列 / 专题合集
- 定时发布
- 修订历史与回滚

### 内容发现

- 归档页
- 标签页
- 分类页
- 系列页
- 相关文章推荐
- 全站搜索
- RSS 与 JSON Feed
- sitemap 与 robots 输出

## 3. 读者互动与管理员身份系统

### 公开访问模型

- 前台阅读默认无需登录
- 公共注册入口已经关闭
- `/register` 现在是说明页，不再创建读者账号

### 评论系统

- 游客评论，公开昵称必填
- 游客邮箱可选，仅用于后续通知，不公开展示
- 隐藏 honeypot 字段用于拦截机器人
- 评论限流与重复评论检测
- 基于白名单 / 黑名单规则的自动审核
- 风险评论进入审核队列
- 楼中回复保持单层结构，阅读更清晰
- 管理员回复默认直接发布
- 管理员可审核、拒绝、删评

### 管理员认证闭环

- 仅管理员可登录
- 登出
- 管理员邮箱验证
- 找回密码
- 重置密码
- 修改密码
- 敏感变更后旧会话失效
- 登录限流与异常登录提醒
- 管理员专属 2FA

### 通知闭环

- 新评论与审核事件会通知管理员
- 游客如果填写邮箱，可收到回复或审核结果邮件
- 管理员私有站内提醒中心位于 `/account/notifications`

## 4. 研究工作流能力

### Papers 模块

- 管理员维护 arXiv 主题
- 手动或定时同步论文
- 公开 Papers 页面
- 管理员私有论文库
- 阅读状态管理（`TO_READ`、`READING`、`COMPLETED`、`ARCHIVED`）
- 阅读进度百分比
- 最近阅读时间
- 批注与高亮摘录

### Weekly Digest

- 基于论文同步结果生成周报
- 公开 Digest 页面
- 后台 Digest 管理
- Digest 引文导出

### 知识连接

- Notes 反向链接
- Citation 文本导出
- Papers / Digest 的 BibTeX 导出

## 5. 后台与运维能力

### 内容管理

- 文章 CRUD
- Notes CRUD
- Journal CRUD
- Gallery CRUD
- Digest 管理界面
- 内容系列管理
- 定时发布字段
- 修订历史恢复流程

### 审核与私有工作台

- 评论审核后台
- 评论规则管理
- 评论命中规则与审核备注查看
- 审计日志
- 管理员账号偏好设置
- 管理员站内提醒与通知偏好设置

### 分析与导出

- 数据分析面板
- 数据导出接口与后台导出页
- 后台总览卡片

## 6. AI 与 RAG 能力

### Provider 管理

- 多模型提供商注册表
- provider 启停与模型参数配置
- 可切换的语音转写 provider
- 用于排查请求链路的 API 校验实验台

### Chat

- 悬浮聊天组件
- 感知当前页面上下文
- 模型切换
- 转写 provider 切换

### RAG

- 手动知识同步
- 混合检索设计
- RAG 管理台
- chunk / source 覆盖与就绪状态查看

## 7. 安全能力

- 自定义会话认证
- HttpOnly Cookie
- 密码重置 token
- 管理员邮箱验证
- 登录限流
- 异常登录提醒
- 管理员 2FA
- 审核与审计控制
- 游客评论反垃圾与规则审核

## 8. 当前刻意保持轻量的地方

虽然功能已经很多，但仍然保持“实用优先”：

- 不再做面向读者的成员区或注册体系
- 角色模型依然保持简单，数据库里保留旧枚举主要是兼容历史数据
- 私有工作台主要面向站长本人，而不是多人协作
- RAG 仍以手动同步为主
- 导出能力更偏备份与迁移，而不是多租户报表系统

## 9. 下一阶段值得增强的方向

后续继续完善时，比较自然的方向包括：

- 更细粒度的通知偏好设置
- 服务端长期运行的后台 worker
- 更适合大文件的对象存储
- 更强的分析归因能力
- 多模态 RAG 入库
